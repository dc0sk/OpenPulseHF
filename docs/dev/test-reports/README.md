# OpenPulseHF Test Reports

This directory contains reports generated by the `openpulse-testmatrix` binary, which runs a
comprehensive signal-path test matrix covering all modulation modes, channel models, FEC
settings, compression variants, and use-case stacks.

## Running the Matrix

```bash
# Quick tier (~30s): all modes × clean/AWGN channels
cargo run --no-default-features -p openpulse-testmatrix

# Full tier (~15min): adds Watterson, Gilbert-Elliott, QRN/QRM/QSB/Chirp
cargo run --no-default-features -p openpulse-testmatrix -- --full

# Via scripts (archives results automatically)
bash scripts/run-test-matrix.sh
bash scripts/run-test-matrix-full.sh
```

## Report Structure

```
docs/test-reports/
├── README.md              — this file
├── latest/                — most recent run (overwritten on each run)
│   ├── summary.md         — pass/fail totals by use case, first-failure list
│   ├── by-mode.md         — mode × channel pass/fail table
│   ├── by-channel.md      — channel × mode pass/fail table
│   ├── by-usecase.md      — full row-per-case table
│   └── raw.json           — machine-readable JSON of all TestResult records
└── archive/               — historical runs (created by run-test-matrix.sh)
    └── YYYY-MM-DDTHHMMSS-<sha>/
```

## Test Dimensions

| Dimension | Values |
|---|---|
| **Modes** | BPSK31, BPSK63, BPSK100, BPSK250, QPSK125, QPSK250, QPSK500, QPSK1000, 8PSK500, 8PSK1000, FSK4-ACK |
| **FEC** | none, interleaved RS (depth 5) |
| **Compression** | none, LZ4 |
| **Channels (quick)** | Clean, AWGN 20dB, AWGN 10dB |
| **Channels (full adds)** | AWGN 5dB/0dB, Watterson Good F1/F2/Moderate/Poor, G-E Light/Moderate/Heavy, QRN, QRM, QSB, Chirp |
| **Use Cases** | RawModem, AdaptiveHPX500, AdaptiveHPX2300, ARDOP, KISS, B2F |

## Interpreting Results

- **PASS** means the received payload exactly matches the transmitted payload (or the
  protocol exchange completed successfully for ARDOP/KISS/B2F cases).
- **FAIL** at high-baud modes in noisy channels is **expected** — the matrix records these
  failures to characterise the mode's noise floor, not to indicate a bug.
- **BER** (bit error rate) is reported for raw-modem cases where a partial decode occurs.
- The quick-tier QPSK1000 failure at AWGN 10dB without FEC is a known physical limitation
  (SNR too low for reliable 3-bits/symbol demodulation without error correction).

## Adding New Test Cases

1. Add a new `UseCase` variant in `apps/openpulse-testmatrix/src/matrix.rs`
2. Implement a runner in `apps/openpulse-testmatrix/src/runners/<name>.rs`
3. Register the runner in `runners/mod.rs::run_case()`
4. Add cases to `cases.rs::build_cases()` under the appropriate tier

New channel models follow the same pattern: add a `ChannelSpec` variant in `matrix.rs`,
a build arm in `channels.rs::build()`, and a label in `ChannelSpec::label()`.
