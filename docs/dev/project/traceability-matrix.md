# Traceability matrix

This is the living, end-to-end traceability matrix for OpenPulseHF. It cross-links two
identifier spaces:

- **REQ-IDs** — requirements, grouped by the categories in `docs/dev/requirements.md`
  (functional `REQ-FUN-*`, physical/radio `REQ-PHY-*`, platform `REQ-PLAT-*`,
  non-functional `REQ-NFR-*`, FEC `REQ-FEC-*`, channel-access `REQ-MAC-*`,
  compression `REQ-CMP-*`, security/trust `REQ-SEC-*`, post-quantum `REQ-PQ-*`,
  regulatory `REQ-REG-*`, competitive performance `REQ-PERF-*`, UX `REQ-UX-*`,
  documentation `REQ-DOC-*`).
- **CAP-IDs** — implemented capabilities (shipped code modules, plugins, services, apps).

Each row of the capability table carries the full chain so a change can be followed in
either direction:

> **Requirement (REQ-ID) → Capability (CAP-ID) → Design decision (+ rationale) →
> Implementation (files/functions) → Tests → Result (actually run) →
> Supporting assets → PRs.**

The Requirements table below is the REQ-ID → CAP-ID view (forward coverage); the
Capabilities table is the CAP-ID → REQ-ID view (backward coverage) plus the rest of the
chain. Test/asset paths are repo-relative. "Latest result" records the most recent
evidence available this session; where it cites `docs/test-reports/latest`, that report is
the testmatrix quick tier at commit `76de87e`, dated 2026-06-29 (555/555 pass, 0 fail).

### Resolved 2026-06-29 — OFDM RawModem failures
The refreshed quick matrix had 7 failures, root-caused to two distinct issues (not a regression):
- **OFDM16 at 10 dB** (3 cases): an SNR-margin issue — OFDM16 had no `mode_min_snr_db` floor (0.0) so it was tested raw at 10 dB where it can't decode; it passes at ≥20 dB and Clean. Fixed by setting OFDM16's floor to 15 dB.
- **OFDM52 with `Rs`/`RsInterleaved`** (4 cases, incl. Clean): the documented padded-frame × 255-byte-RS-block limitation — the demod emits a padded byte count RS reads as a corrupted block, so it fails even on a clean channel (`None` passes everywhere). Fixed by restricting plain OFDM52 to `FecMode::None` in the raw-modem multicarrier section (`OFDM_RAW_FRAMING_ONLY` in `apps/openpulse-testmatrix/src/cases.rs`); OFDM52 stays covered via `None` + the OFDM-HOM/soft-FEC sections, and the coverage regression test still passes (no silent exclusion).

Re-run quick matrix: **555/555 pass, 0 fail** (commit `76de87e`, 2026-06-29).
